# XY64 — design

The 2D sibling of Sliders64: the whole 8×8 grid is a single XY surface.
Roadmap milestone 1; design refined during the July 2026 product review.

## Interaction

- **Press a pad** to set the target. The **cursor glides** toward it at the
  slew rate selected on the scene buttons (the Sliders64 idiom: A instant …
  H slowest, same rate table, default C = 0.5 s full range).
- The target pad is shown **dim**, the gliding cursor **bright**; when the
  cursor arrives they merge into one bright pad.

## Decisions

- **The cursor is continuous, not pad-quantized.** Pads only quantize the
  *targets* (8×8 destinations); the output is the cursor's continuous
  position, so a glide sweeps smoothly through the whole range — the feared
  stair-stepping of an 8-step surface never appears. Only the LED display
  rounds to the nearest pad.
- **Straight-line vector glide**, constant speed in normalized space, rather
  than independent per-axis slew: the cursor visibly travels *toward the pad
  you pressed*, which is the whole gestural promise of an XY surface. With
  per-axis slew the path bends when one axis lands first.
- **TRIG output — "the cursor lands".** One trigger per press, fired at
  arrival. At instant slew (scene A) arrival is immediate, so XY64 doubles as
  a trigger pad with position CV — the module is an event source, not only a
  modulation source, at the cost of one jack.
- Momentary/held pads have no special meaning (no drag: the Launchpad Mini
  sends no polyphonic aftertouch or slide); release is ignored.
- Glide continues while the page is inactive — XY64 is a CV source, like
  Sliders64's slews which also keep moving in the background.

## I/O

| Jack | Meaning |
|---|---|
| X out | cursor X, 0 V (left) … 10 V (right) |
| Y out | cursor Y, 0 V (bottom) … 10 V (top) |
| TRIG out | 1 ms trigger when the cursor reaches the target |

Menu: cursor color, target color. Serialized: cursor, target, slew index,
colors.
